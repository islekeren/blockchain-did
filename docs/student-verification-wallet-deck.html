06 / ADMIN

Admin/Governance controls trust.

Registry authority

Keeps public trust changes explicit and auditable.

Trusted issuer registryAdd university wallet + DID, remove trust when needed.
Schema registryRegister StudentCredential schema hash and readable name.
Owner-only writesPrototype uses one owner account for governance actions.
Input validationReject zero issuer address, empty DID, and zero schema hash.
Audit trailKeep admin actions visible in local audit/history screens.
Production pathReplace single owner with multi-sig, DAO, or public authority.

Student lifecycleCreate student records and update active/inactive status.

08 / HOLDER

Student controls presentation.

Student/Holder

Wallet-like approval flow before any verifier receives proof.

Credential walletStudent sees issued, expired, pending, and revoked credentials.
Request reviewShows verifier name, credential type, expiry, and shared fields.
Consent stepNothing is sent until the holder approves the request.
Presentation proofWallet signs request id, nonce, credential hash, and holder address.
Privacy boundaryVerifier receives active-student proof, not full university documents.
Production UXMetaMask is prototype UX; wallet complexity can be hidden later.
